What This System Does

Warren and Billy have a daily video call on Google Meet where they discuss multiple client projects — materials, measurements, scope changes, timelines. Gemini automatically records and transcribes these calls.

This system takes those transcripts, identifies which clients were discussed, splits the transcript into per-client segments, and saves each segment to the correct client's Google Drive folder — automatically, with zero manual handling.

The saved transcripts feed into the Trellis quoting system, giving Claude the rich context it needs to draft accurate quotes.

→Warren + Billy video call on Google Meet (Gemini records + transcribes)
→Gemini auto-emails transcript to hello@saltlandscaping.com.au
→Apps Script runs every 15 min — detects the new email
→Extracts full transcript from the linked Google Doc (Transcript tab)
→Matches client mentions against Drive folder names (EXACT → NAME → ADDRESS)
→Splits transcript into per-client segments
→Saves each segment to client's 06 - Quoting Context subfolder in Drive
→Unmatched segments → Billy gets email with "Assign Client" button
→Trellis / Claude references the quoting context folder when drafting quotes

Prerequisites

  • Google Workspace account: hello@saltlandscaping.com.au
  • Google Meet with Gemini note-taking enabled on the workspace
  • A recurring Google Calendar invite called Warren + Billy - Quoting (Tue-Fri)
  • Client project folders in Google Drive under Landscaping Projects
  • Folder naming convention: {Client Name} - {Address} (matches HubSpot deal names)

Google Drive Structure

// The parent folder that contains all client project folders Landscaping Projects/ Sean Bryant - 42 Marine Pde Capalaba/ 01 - Overview/ 02 - Communications/ 03 - Site Information/ 04 - Designs/ 05 - Proposals/ 06 - Quoting Context/ // Created automatically Warren-Billy Quoting Notes - 2026-04-14.gdoc Warren-Billy Quoting Notes - 2026-04-15.gdoc Adam Horton - 15 Valley View Rd Valdora/ 01 - Overview/ ... 06 - Quoting Context/ Warren-Billy Quoting Notes - 2026-04-14.gdoc // For segments that can't be matched to a client Unmatched Transcripts/ Unmatched - Notes 'Warren + Billy - Quoting' - 2026-04-14.gdoc
The 06 - Quoting Context subfolder is created automatically the first time a transcript is saved to a client folder. No manual setup required for any client.

How Client Matching Works

Billy announces each client before discussing their project on the call. The system matches what Billy says against the Google Drive folder names using a 3-tier cascade:

PriorityStrategyExampleWhat Triggers It
1 EXACT Billy says "Sean Bryant - 42 Marine Pde Capalaba" Full folder name found verbatim in transcript
2 NAME Billy says "Sean Bryant" (address garbled by Gemini) Client full name found with word boundaries
3 ADDRESS Billy says "the Capalaba job on Marine Parade" 2+ address tokens match, or 1 long suburb name (6+ chars)
4 UNMATCHED Nothing matches any folder Billy gets email notification to manually assign
For best accuracy: Billy should say the client name and address in the same format as the Drive folder / HubSpot deal before discussing each project. e.g. "Okay, let's talk about Sean Bryant - 42 Marine Parade Capalaba."

Unmatched Segment Workflow

When a transcript segment can't be matched to any client folder:

  1. The segment is saved to an "Unmatched Transcripts" folder in Drive
  2. Billy receives a SALT-branded notification email with a preview of the unmatched text
  3. The email contains an "Assign Client" button
  4. Clicking it opens a searchable list of all client folders (web app)
  5. Billy taps the correct client → the document is automatically moved to that client's 06 - Quoting Context folder
No manual Drive navigation required. Billy just reads the preview, taps the client name, done.

Setup Guide: Building From Scratch

1 Create the Google Apps Script Project
  1. Go to script.google.com while signed in as hello@saltlandscaping.com.au
  2. Click New project
  3. Name it "SALT Transcript Processor"
  4. Select all default code (Cmd+A) and delete it
  5. Paste the full script from ~/Projects/salt-transcript-processor/Code.gs
  6. Click Save (Cmd+S)
2 Verify Configuration

Check these values at the top of the script match your environment:

PARENT_FOLDER_ID: '1GS6Z6vPwqUmFK1LbYgaUFhB1BTRPdeyz' // "Landscaping Projects" folder ID GMAIL_ACCOUNT: 'hello@saltlandscaping.com.au' GEMINI_SENDER: 'gemini-notes@google.com' MEETING_TITLES: ['Warren + Billy', 'Warren + Billy - Quoting'] NOTIFICATION_EMAIL: 'billy@cactusculture.com.au' CLIENT_SUBFOLDER: '06 - Quoting Context'

To find a Google Drive folder ID: open the folder in Drive, the ID is the last part of the URL after /folders/.

3 Run Setup
  1. In the function dropdown (top of editor, next to Run button), select setup
  2. Click Run
  3. Google will ask you to authorize permissions — review and approve all
  4. Check the execution log to confirm:
    • Gmail label "Processed-Transcript" created
    • Log spreadsheet "Transcript Processing Log" created
    • "Unmatched Transcripts" folder created
    • 15-minute trigger created
4 Deploy as Web App

This enables the "Assign Client" feature for unmatched transcripts.

  1. Click Deploy (top right) → New deployment
  2. Click the gear icon next to "Select type" → choose Web app
  3. Set Execute as: Me (hello@saltlandscaping.com.au)
  4. Set Who has access: Anyone within Salt Landscaping
  5. Click Deploy
  6. A URL will be generated — the notification emails will use this automatically
5 Set Up the Calendar Invite
  1. Create a recurring Google Calendar event (Tuesday to Friday)
  2. Title: Warren + Billy - Quoting (must match exactly)
  3. Add Warren and Billy as attendees
  4. Use Google Meet for the video call
  5. Ensure Gemini note-taking is enabled for the workspace
Critical: The calendar invite title must exactly match one of the values in the MEETING_TITLES config. If it doesn't match, the script will ignore the transcript.

Billy's Meeting Protocol

For the system to work accurately, Billy should follow this simple pattern during every Warren + Billy call:

Before discussing each client

Clearly say the client name and address in the same format as the Drive folder:

"Okay, let's talk about Sean Bryant - 42 Marine Parade Capalaba." // Then discuss materials, measurements, scope, timeline, etc. "Alright, next is Adam Horton - 15 Valley View Road Valdora." // Discuss Adam's project...

Monitoring & Troubleshooting

Log Spreadsheet

A spreadsheet called "Transcript Processing Log" is created in the Drive root. Each row shows:

ColumnDescription
TimestampWhen the action occurred (AEST)
Email SubjectThe Gemini email subject line
Message IDGmail message ID for debugging
ActionPROCESSING, SAVED, NO_MATCH, MANUAL_ASSIGN, FATAL_ERROR, etc.
Client MatchedWhich client folder the segment was matched to
Folder PathFull path where the doc was saved
Doc URLDirect link to the created Google Doc
NotesMatch type, character count, error details

Common Issues

ProblemCauseFix
Script doesn't pick up transcripts Meeting title doesn't match config Check calendar invite title matches MEETING_TITLES exactly
Client not matched Billy didn't say the client name clearly, or Gemini garbled it Billy will get the "Assign Client" email — tap to assign manually
Wrong client matched Similar names or common words causing false positives Check the log sheet. Move the doc manually in Drive if needed
Duplicate docs Script ran twice on same email Built-in: checks for existing doc with same name before creating
"Assign Client" link doesn't work Web app not deployed or redeployed Go to Apps Script → Deploy → Manage deployments → verify active
No transcript content (only summary) Google Doc doesn't have a Transcript tab Falls back to email body. Ensure Gemini transcription is enabled

Execution Limits

  • Google Apps Script: 6 minutes max execution time per run
  • Config limits processing to 10 emails per run (safety valve)
  • If you have a backlog of 50+ unprocessed emails, they'll be processed over several trigger runs
